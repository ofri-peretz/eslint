# `docs/` — Repo Documentation Index

Internal repository documentation. User-facing rule docs live under `apps/docs/` (the Fumadocs site at `eslint.interlace.tools`); this folder is for contributors, agents, and reviewers working on the repo itself.

## Standards & Policies

- [`QUALITY_STANDARDS.md`](./QUALITY_STANDARDS.md) — production-ready checklist for ESLint rules (the gate every rule must pass before release).
- [`ESLINT_VERSION_SUPPORT.md`](./ESLINT_VERSION_SUPPORT.md) — which ESLint majors we support, the 20% gate, the forward-looking exception, and how to refresh the data (`npm run stats:eslint-versions`).
- [`META_HYGIENE.md`](./META_HYGIENE.md) — rule-meta completeness audit. Generated by `npm run audit:meta` (tsx scripts/audit-rule-meta-completeness.ts); surfaces fields the formatter expects.

## Process & Workflow

- [`PLUGIN-REVIEW-WORKFLOW.md`](./PLUGIN-REVIEW-WORKFLOW.md) — step-by-step review process for any plugin (used during release reviews).
- [`SDK_COMPATIBILITY_OVERVIEW.md`](./SDK_COMPATIBILITY_OVERVIEW.md) — system overview for the SDK compatibility tests in `.github/workflows/sdk-*.yml` (weekly checks against latest SDK versions).
- [`SDK_COMPATIBILITY_TEMPLATE.md`](./SDK_COMPATIBILITY_TEMPLATE.md) — template + pattern doc for adding a new SDK compat test.

## Integrations

- [`oxlint-integration.md`](./oxlint-integration.md) — two-tier lint pipeline architecture (oxlint primary, ESLint for type-aware/complex rules). Status: experimental.

## Long-form material

- [`roadmaps/`](./roadmaps/) — multi-quarter rule and feature roadmaps (OWASP LLM 2025, OWASP Agentic 2026).
- [`research/`](./research/) — reference notes and source material that backs the roadmaps.
