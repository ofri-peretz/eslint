# Use Interlace with Cursor

> **Time to set up:** ~3 minutes.
> **Result:** Cursor's agent can call `audit_file`, `find_rule_violations`, `suggest_fix`, and `list_rules` against any of the 11 Interlace plugins as typed MCP tools. Identical tool surface to [Claude Code](./claude-code).

## Step 1 — install the CLI + the plugin servers

```bash
npm install -g @interlace/cli
# Pick which plugins to expose:
npm install -g @interlace/secure-coding-mcp \
              @interlace/browser-security-mcp \
              @interlace/node-security-mcp
# (Or rely on npx — no install needed; see step 2.)
```

## Step 2 — register the servers with Cursor

Cursor's MCP config lives at `~/.cursor/mcp.json` (global) or `<project>/.cursor/mcp.json` (project-scoped).

```json
{
  "mcpServers": {
    "secure-coding":   { "command": "npx", "args": ["--yes", "@interlace/secure-coding-mcp"] },
    "node-security":   { "command": "npx", "args": ["--yes", "@interlace/node-security-mcp"] },
    "browser-security":{ "command": "npx", "args": ["--yes", "@interlace/browser-security-mcp"] }
  }
}
```

The full list (each is `@interlace/<plugin>-mcp`):

`secure-coding` · `browser-security` · `node-security` · `crypto` · `jwt` · `express-security` · `lambda-security` · `mongodb-security` · `nestjs-security` · `vercel-ai-security` · `pg`

## Step 3 — restart Cursor

The MCP servers will appear in Cursor's tool panel. Each exposes `list_rules`, `audit_file`, `find_rule_violations`, `suggest_fix`.

## Step 4 — use it

In Cursor's agent panel:

> *"Use the secure-coding MCP server to audit the file I just changed. Apply auto-fixes for any high-confidence findings."*

Cursor calls `audit_file` → reads findings → calls `suggest_fix` per high-confidence finding → applies via its built-in editing tools.

Other prompts that work well:

| Prompt | What Cursor's agent does |
| :--- | :--- |
| *"Show me every hardcoded credential in src/"* | Calls `find_rule_violations` with rule `no-hardcoded-credentials` |
| *"Audit the diff I'm about to commit"* | Walks the changed files, audits each, summarizes |
| *"What rules does the jwt MCP server have?"* | Calls `list_rules` on the jwt server |

## Tool surface

Identical to the [Claude Code walkthrough](./claude-code#what-each-tool-returns) — same JSON shapes, same semantics, same `confidence`-aware fix routing.

## Why use this in Cursor specifically

Cursor's agent already has good code-editing tools — `suggest_fix` returning a structured `{range, replacement}` patch lets the agent apply Interlace fixes through Cursor's existing Edit primitives without any string parsing. Auto-fix loops become reliable.

The `confidence` field on every finding means Cursor can:

- **`high`-confidence** → apply the fix automatically
- **`medium`** → propose the diff and wait for your review
- **`low`** → flag for manual remediation

The bands are calibrated by the [ILB-Confidence bench](https://github.com/ofri-peretz/eslint/tree/main/benchmarks/suites/ilb-confidence) — empirical precision per band, not vendor vibes.
