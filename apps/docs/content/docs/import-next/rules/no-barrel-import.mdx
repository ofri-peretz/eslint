Disallow imports from barrel files to improve build performance and tree-shaking

## Rule Details

Barrel files (index.js/ts) that re-export from multiple modules can significantly impact build performance and prevent effective tree-shaking. This rule detects imports from known barrel patterns and suggests direct imports instead.

## Options

| Option              | Type       | Default | Description                                               |
| ------------------- | ---------- | ------- | --------------------------------------------------------- |
| `knownBarrels`      | `string[]` | `[]`    | Regex patterns for paths known to be barrels              |
| `ignoreNodeModules` | `boolean`  | `true`  | Ignore imports from node_modules                          |
| `ignoredPackages`   | `string[]` | `[]`    | Packages to ignore (those with good tree-shaking support) |

### Configuration Example

```javascript
// eslint.config.js
export default [
  {
    rules: {
      'import-next/no-barrel-import': [
        'error',
        {
          knownBarrels: ['/components/index', '/utils/index'],
          ignoreNodeModules: true,
          ignoredPackages: ['lodash-es', 'date-fns'],
        },
      ],
    },
  },
];
```

## Known False Negatives

<FalseNegativeCTA />

This rule uses pattern-based detection. The following may not be caught:

- **Dynamic patterns** - Runtime-generated code or values
- **Indirect references** - Values passed through multiple variables or functions
- **External data** - Data from APIs, databases, or user input
