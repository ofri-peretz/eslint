## The Interlace Workflow

Every security vulnerability follows a **3-step journey** from detection to remediation. Interlace is designed to make each step seamless‚Äîfor both humans and AI agents.

<Steps>

### Step 1: Detect

ESLint catches the vulnerability in your code, highlighting the exact line and providing context.

### Step 2: Understand

Structured metadata (CWE, OWASP, CVSS) gives you‚Äîand your AI assistant‚Äîeverything needed to understand the risk.

### Step 3: Fix

Verified remediation patterns guide the fix, ensuring you apply the correct solution every time.

</Steps>

---

## Workflow in Action

Let's walk through a real SQL Injection vulnerability:

### üî¥ Step 1: Detect

Your editor shows a red squiggly on line 19:

```js {3}
async function getUserById(userId: string) {
  // ‚ùå ESLint Error: SQL query constructed via string concatenation
  const query = "SELECT * FROM users WHERE id = " + userId;
  return db.query(query);
}
```

The ESLint rule `secure-coding/no-sql-concatenation` has detected the vulnerability.

---

### üü° Step 2: Understand

Hovering over the error reveals rich metadata:

| Field      | Value                    | Meaning                 |
| ---------- | ------------------------ | ----------------------- |
| **CWE**    | CWE-89                   | SQL Injection           |
| **OWASP**  | A03:2021                 | Injection category      |
| **CVSS**   | 9.8                      | Critical severity       |
| **Impact** | Data breach, auth bypass | Real-world consequences |

<Callout type="warn" title="Why This Matters">
  A CVSS of 9.8 means this vulnerability could allow complete database access.
  This isn't a "fix later" issue‚Äîit needs immediate attention.
</Callout>

---

### üü¢ Step 3: Fix

The rule provides a **verified fix pattern**:

```js {3-6}
async function getUserById(userId: string) {
  // ‚úÖ Fixed: Parameterized query prevents injection
  const result = await db.query(
    'SELECT * FROM users WHERE id = $1',
    [userId]
  );
  return result.rows[0];
}
```

The fix:

- Uses parameterized queries (placeholders like `$1`)
- Separates data from SQL syntax
- Prevents any SQL injection regardless of input

## How AI Agents Use This Workflow

When you ask an AI assistant to fix the error:

1. **Detect**: AI reads the ESLint error output
2. **Understand**: AI parses the structured metadata (CWE-89, CVSS 9.8)
3. **Fix**: AI applies the documented remediation pattern

<Callout type="success" title="No Hallucinations">
  Because the AI has structured context, it applies the **exact fix
  pattern**‚Äînot a guess. This is the power of AI-native documentation.
</Callout>

| Aspect            | Traditional Linter       | Interlace                     |
| ----------------- | ------------------------ | ----------------------------- |
| **Detection**     | "Possible SQL injection" | Exact line, node, and pattern |

```yaml
      echo "Critical security issues found!"
      exit 1
    fi
```
