# `tools/` — Built / Published Tooling

Code that gets *built*, *published*, or *consumed at runtime* by packages in this monorepo. This is distinct from [`../scripts/`](../scripts/), which holds repo-wide automation invoked from `npm run` or CI.

> **Boundary rule:** if it's imported, packaged, or part of the lint pipeline, it lives here. If it's invoked from `npm run` or a workflow, it lives in `../scripts/`.

## Subdirectories

| Subdir | Purpose | Status |
| :--- | :--- | :--- |
| [`cwe-analytics-engine/`](./cwe-analytics-engine/) | The `@interlace/cwe-analytics-engine` published npm package — CWE coverage analytics for ESLint plugins. Has its own `package.json`, tests, and CHANGELOG. | Published |
| [`eslint-rules/`](./eslint-rules/) | Custom ESLint rules consumed by this repo's own `eslint.config.mjs` (e.g. `changelog-format`, `require-vitest-watch-false`). Not published — internal-only. | Internal |
| [`oxlint-plugins/`](./oxlint-plugins/) | Oxlint shim files mirroring our ESLint plugins. Generated by [`../scripts/generate-oxlint-shims.mjs`](../scripts/generate-oxlint-shims.mjs). | Generated |
| [`scripts/`](./scripts/) | Tools-specific automation (changelog formatting/repair). Distinct from `../scripts/` — these are scoped to the changelog workflow and shouldn't be confused with repo-wide automation. | Internal |

## Conventions

- Subdirs that publish to npm have their own `package.json`, `README.md`, `CHANGELOG.md`, and `LICENSE`.
- Internal-only subdirs may skip those files but should keep a brief top-of-file comment explaining purpose.
- Generated subdirs (`oxlint-plugins/`) shouldn't be hand-edited — re-run the generator instead.
