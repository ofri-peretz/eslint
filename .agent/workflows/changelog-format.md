# Changelog Format Standard

All `CHANGELOG.md` files across the monorepo MUST follow the [Keep a Changelog](https://keepachangelog.com/en/1.0.0/) format. This is enforced by the `local/changelog-format` ESLint rule.

## Required Header Format

```
## [version] - YYYY-MM-DD
```

### ✅ Correct Examples

```markdown
## [8.3.0] - 2026-02-08

## [1.0.0] - 2025-12-31

## [Unreleased]
```

### ❌ Wrong Examples (Nx Release default format)

```markdown
## 8.3.0 (2026-02-08) ← missing brackets, wrong date format

## [8.3.0] (2026-02-08) ← parentheses instead of dash

# 8.3.0 (2026-02-08) ← wrong heading level
```

## Key Rules

1. **Version MUST be wrapped in brackets**: `[1.0.0]` not `1.0.0`
2. **Date separated by dash**: `- 2026-02-08` not `(2026-02-08)`
3. **Heading level must be `##`**: Not `#` or `###`
4. **Date format**: `YYYY-MM-DD` (ISO 8601)
5. **`# Changelog` title is allowed** as the first line (it's skipped by the lint rule)
6. **`## [Unreleased]`** is a valid header

## Nx Release Generates Wrong Format

Nx Release generates headers in `## version (YYYY-MM-DD)` format by default. After any Nx Release, you MUST run the fix script:

```bash
// turbo
node tools/scripts/fix-changelog-format.js
```

This converts all `eslint-plugin-*` changelogs from `## version (date)` → `## [version] - date`.

> **Note**: The fix script only processes `eslint-plugin-*` packages. For `eslint-devkit` and `cli`, manually ensure the format is correct.

## When Adding Changelog Entries Manually

When documenting new features, fixes, or changes manually, always use the correct format:

```markdown
## [8.3.0] - 2026-02-08

### Added

- **rule-name**: Description of the rule (CWE-XXX, CVSS X.X)

### Fixed

- **rule-name**: Description of the fix

### Changed

- **rule-name**: Description of the change
```

## Validation

The format is enforced by:

- **ESLint rule**: `local/changelog-format` (defined in `tools/eslint-rules/changelog-format.js`)
- **CI/CD**: Runs `nx lint` which checks all `CHANGELOG.md` files
- **Fix script**: `tools/scripts/fix-changelog-format.js` auto-corrects Nx Release format

## Subsection Headers

Use standard Keep a Changelog categories:

- `### Added` — new features
- `### Changed` — changes in existing functionality
- `### Deprecated` — soon-to-be removed features
- `### Removed` — removed features
- `### Fixed` — bug fixes
- `### Security` — vulnerability fixes

## Important Notes

- The `### ⚠️ BREAKING CHANGES` header is also valid for breaking change sections
- The `### ❤️ Thank You` section is auto-generated by Nx Release and is fine
- `### Bug Fixes` (Nx Release default) is acceptable as it doesn't trigger the lint rule (only `##` headers are checked)
