## Bug found and fixed during this audit

While re-running benches, ILB-Wild started showing **0 findings on every repo**. Root cause: ESLint v9 silently ignores files outside the `cwd` setting, and the worker template hardcoded `cwd: ROOT` (= the eslint repo root). After moving target clones to `~/repos/ofriperetz.dev/oos/` (outside the eslint repo), all target files were silently skipped.

**Fix:** [`scripts/ilb-wild.mjs:594`](../scripts/ilb-wild.mjs) now sets `cwd: path.dirname(BENCH_DIR)` — the closest common ancestor of the eslint repo (where plugins live) and the target repo (under `oos/`). After the fix, wild reports valid 7,439 findings across 22 repos, 4.08/kLoC density. **Without this fix the audit would have used 0-finding data.** Worth flagging because it almost ruined this audit.

## Prioritized action items

Each is shovel-ready: the rule + concrete condition to add. No vague "investigate."

### P0 — fix one rule, drop FPs by 60%

**`secure-coding/detect-object-injection`** — refine context exemptions. Current rule fires on any `obj[key]` where `key` is not a literal. Add exemptions for:

1. **Typed-array indexing.** If `obj` resolves to `Float32Array` / `Uint8Array` / `Int32Array` / etc., skip. (Catches the Three.js render loop case.)
2. **`for..in` / `Object.keys` / `for..of` iteration.** When `key` comes from a known-safe iterator, skip.
3. **`hasOwnProperty` / `Object.hasOwn` guards.** When access is preceded by `if (key in obj)` / `Object.hasOwn(obj, key)`, skip.
4. **AST-walker contexts.** When file imports `@babel/types`, `@babel/traverse`, `recast`, `jscodeshift`, `eslint`, or `estree-walker`, treat `node[key]`-style access as safe.
5. **Test files.** `*.test.*`, `*.spec.*`, `**/__tests__/**`, `**/__fixtures__/**`.

**Impact:** removes 2,468 of 4,106 ILB-Edge candidates (60%).

### P0 — fix one rule, drive Juliet F1 from 76.5% → ~88%

**`pg/no-floating-query`** — recognize parameterized and prepared-statement patterns as **the correct pattern**, not a violation. Current rule fires on any `pool.query(sql)` regardless of how `sql` was constructed. Add:

1. **Skip when 2nd arg is an array.** `pool.query(sql, [$1Args])` is the parameterized form — never flag.
2. **Skip prepared-statement objects.** `{ name, text, values }` form — never flag.
3. **Skip when `sql` is a literal string with no concatenation.** Static queries are safe.

**Impact:** removes 3 Juliet FPs (37.5% of the 8). Juliet F1 76.5% → ~88%, BAS 38.5% → ~62%.

### P0 — fix the ILB-Arena 1-FP

**`lambda-security/no-unvalidated-event-body`** — scope to Lambda handler context. Currently fires on any function that touches `headers`. Add:

1. **Detection:** function's first parameter is named `event` AND second is named `context`/`callback`, OR function is exported as `handler`, OR file imports `aws-lambda` types.
2. **Don't fire** on express handlers that read `req.headers`.

**Impact:** drops Arena FPs from 1 to 0. **F1 → 100% on the synthetic head-to-head.**

### P1 — fix the quality fleet's noisiest rule

**`reliability/no-missing-null-checks`** — 73 of 118 quality FPs come from this single rule (62%). Add:

1. **TypeScript narrowing recognition.** If `x` was just narrowed by `if (x)` / `if (x !== null)` / typeof guard, skip.
2. **Optional-chaining recognition.** `x?.y` is itself a null check; don't demand another.
3. **Prior-check flow analysis.** Walk back to find guards in the same scope.

**Impact:** quality FPs drop from 31 fixtures to ~20. F1 64.8% → ~78%.

### P1 — long-tail Juliet fixes

Fix three more rules to drive Juliet F1 toward 95%:

- **`node-security/no-zip-slip`** — scope to `unzipper`/`tar`/`adm-zip` extraction context only. (Removes 2 Juliet FPs including the wrong-domain one on `execFile-array.js`.)
- **`secure-coding/no-improper-sanitization`** — recognize literal-string innerHTML as safe. (Removes 1 Juliet FP.)
- **`secure-coding/no-hardcoded-credentials`** — skip when value is a label/identifier (`'password-label'`, `'token-field'`). (Removes 1 Juliet FP + ~280 webpack FPs.)

**Combined impact:** Juliet F1 → ~95%; webpack FPs drop ~280.

### P1 — quality fleet refinements

- **`reliability/no-unhandled-promise`** — recognize outer try/catch. (Removes 26 quality FPs.)
- **`maintainability/consistent-function-scoping`** — recognize closure-intent functions. (Removes 8 quality FPs.)
- **`reliability/require-network-timeout`** — recognize AbortController patterns. (Removes 5 quality FPs.)

**Combined impact:** Quality F1 → ~85%, beating jsdoc's 66.1% and putting us clearly #1.
