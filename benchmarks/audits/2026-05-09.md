## 1. What landed

### ILB-Formatter (new bench, #10)

Whole-run formatter benchmark suite. Measures `@interlace/eslint-formatter`
against ESLint built-ins (`stylish`, `json`, `json-with-metadata`, `html`)
across **6 corpus shapes × 5 scales × 8 formats = 240 cells** per run on
the **Cost / Effectiveness / Latency** triad.

| Mode | Cost vs `eslint-stylish` | Effectiveness | Latency P50 mean |
| :--- | ---: | ---: | ---: |
| `interlace-compact` | **−89.5 %** | 4.0 / 4 | 0.04 ms |
| `interlace-ndjson` *(v1.1)* | **−66.7 %** | 4.0 / 4 | 0.06 ms |
| `interlace-json` | −67.9 % | 4.0 / 4 | 0.06 ms |
| `interlace-human` | −62.3 % | 4.0 / 4 | 0.08 ms |

Headline finding on `mono-rule-storm/extreme` (one rule firing 6,000 times
across 2,000 files): `interlace-compact` is **77× faster and ~1,400× cheaper
in tokens** than `eslint-stylish` at the same signal score (4 / 4).
`eslint-html` drops the `fixable` flag on every all-fixable fixture
(3.83 / 4 mean) — a real upstream signal-loss surface in a comparator we
don't own.

Three contracts gate every run, all passing at v1.1:
- **Signal contract** — every structured format scores 4 / 4 on every
  fixture (`ruleId`, `severity`, `count`, `fixable` recoverable).
- **Latency contract** — `interlace-*` P50 ≤ {tiny:5, small:10, medium:25,
  large:50, extreme:250} ms per scale.
- **Regression check** — every cell within `max(+5 %, +10 tok)` cost,
  `max(+50 %, +0.5 ms)` latency, signal score ≥ baseline, vs the
  snapshotted [`baseline.json`](../results/ilb-formatter/baseline.json).
  Subjects-under-test (`interlace-*`) gate the build; comparator drift
  (`eslint-*`) is observational.

Methodology: [`suites/ilb-formatter/methodology.md`](../suites/ilb-formatter/methodology.md) (v1.1).

### Cost / Effectiveness / Latency vocabulary contract

Lifted to the [README](../README.md) §1 as a peer of the FP/FN philosophy.
Every result JSON now must publish all three aspects with the standard
field names (`tokensO200k`, `signalScore`, `latencyMsP50`, etc.) — no
synonyms, no per-bench renames. The scorecard's bench table reorganised
from a generic "Score" column to three explicit columns so a missing
aspect is visible rather than smoothed-over.

### Provenance contract

Added as a peer of the triad. Every result JSON answers two questions a
sceptical reader always asks: *which model produced or scored the output*
(`provenance.model.kind ∈ {llm, tokenizer-proxy, none}`) and *which tools
were involved at which versions in what role* (`provenance.tools.{count,
items[]}` with name + version + role per tool). The runner asserts
`tools.count === items.length` and aborts on drift; the validator
([`scripts/ilb-validate-results.mjs`](../../scripts/ilb-validate-results.mjs))
now enforces the same shape on every result.

ILB-Formatter publishes 5 tools per run: `@interlace/eslint-formatter`
(subject), `eslint` (comparators), `gpt-tokenizer` (cost-axis surrogate),
`node:process.hrtime.bigint` (latency-axis), `tsx` (TS runtime).

### Formatter package improvements

- **Severity-first ordering** — errors before warnings, then count desc,
  then ruleId asc. Prevents a rare critical error from being buried
  under noise (the "lost in the middle" effect).
- **`ndjson` mode** (`ESLINT_FORMAT_MODE=ndjson`) — newline-delimited
  JSON for streaming agent loops. Summary line ships first; one self-
  describing JSON per rule line.
- **JSON: `summary` ships before `rules`** — Anthropic prompt-cache
  prefixes match by prefix, so the stable summary at the front
  survives even when the per-rule list churns
  ([Anthropic guide](https://platform.claude.com/docs/en/build-with-claude/prompt-caching);
  up to 90 % input cost / 85 % latency reduction on cache hits).
- **Char-budget knob** (`ESLINT_FORMAT_CHAR_BUDGET=N`) — hard ceiling
  on output size; trims lowest-priority rules until the render fits,
  with an explicit `[truncated N rule(s)]` notice. Severity-first sort
  means errors survive first.
- **Per-message text + `nodeType` + `suggestions[]` round-trip** —
  previously dropped, now surfaced in every mode.
- **CWE / CVSS surfaces** — rendered inline in human / compact / json /
  ndjson when `meta.docs.cwe` / `meta.docs.cvss` are present.
- **ANSI color in human mode** — auto-disabled when `!process.stdout.isTTY`
  or `NO_COLOR` is set; honors `FORCE_COLOR`.
- **Mode-aware `MAX_LOCATIONS` cap** — compact / human at 5,
  json / ndjson at 50.
- **JSON Schema published** at
  [`packages/eslint-formatter/schema.json`](../../packages/eslint-formatter/schema.json)
  — downstream agents validate they're consuming the envelope correctly.

### CI gate wiring

[`.github/workflows/benchmark.yml`](../../.github/workflows/benchmark.yml)
gains two new jobs that run on every matching PR:

- `ilb-formatter` — builds the formatter, runs all 40 unit + snapshot +
  ESLint-integration tests, runs ILB-Formatter (240 cells, 3 contracts),
  validates the result envelope. 5-minute timeout. Hard gate.
- `audit-meta` — runs `npm run audit:meta`, uploads
  `docs/META_HYGIENE.md` as a workflow artifact. Read-only today;
  promote to a hard gate once all plugins reach a target completeness
  threshold.

### Validator improvements

`ilb:validate-results` was flagging legitimate `cost` / `effectiveness` /
`latency` blocks as "blocked synonyms" because the schema's
`synonymsBlocked.blockedAtRoot` list overlaps with its own valid-property
names. Fixed: only flag when the value is primitive (the synonym shape);
allow when the value is an object (the structured block shape) and let
`checkSubsetKeys` enforce the inner allow-list. Dropped repo-wide fatal
issue count from **14 → 9**.

Also added `checkProvenance` enforcing the contract above.
